Project information

For the course project hosting I together with 5 fellow students are creating our own hosting platform that is meant to run other students PHP projects. To control the platform we also have to create an API ourselves. We are currently working on the project.

Project Hosting

We already have a schema that shows what our platform will look like. We have an Admin VM in which we want to run different services. One of these is Zabbix, which is our monitoring software. We are also going to run an scp server on it so that the user can put his/her files on our platform in a safe and encrypted way. Our API will also run on the Master VM, just like our MySQL database that contains the data about our users. We are also going to use PHP MyAdmin to get a nice dashboard for our database. Finally, our Admin VM will also become the Puppet master with which we will set up and control all the other VMs. In addition to the Admin VM, we also have 2 other VMs running for our customers. These are both puppet clients in each of which 2 kubernetes nodes will be configured in a cluster to run all our services in separate containers. For our customers we need to provide PHP, MySQl and an apache web server that all come in a seperate container in each node. To run a mailcow container we are also going to provide one node where we run a mailcow container so that all our customers can use it to send automatic mails because this was one of the requirements they gave us.
