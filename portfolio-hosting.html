Project information

For this course I have to create my own hosting platform that is meant to run other students' PHP projects. The platform is completely controlled by an API that we also have to create ourselves. I am doing the project together with 5 other students.

Project Hosting

We currently already have a schedule that shows what our platform will look like. We have an Admin VM in which we want to run different services: one of them is Zabbix, which is our monitoring software, we are also going to run a scp server in it so that the user can put his files on our platform safely and encrypted. Our API will also run on the Master VM, just like our MySQL database that contains the data about our users. We are also going to use PHP MyAdmin to get a nice dashboard for our database. Finally, our Admin VM will also become the Puppet master with which we will set up and control all the other VMs. In addition to the Admin VM, we also have 2 other VMs running for our customers. These are both puppet clients and here 2 kubernetes nodes will run on each VM because we are also going to set up a kubernetes cluster to run all our services in a separate container. For our customers we need to provide PHP, MySQl and an apache web server. So these all come in a separate container for each node. We are also going to provide one node where we run a mailcow container so that all our customers can use it to send automatic mails because this was one of the requirements they gave us.
